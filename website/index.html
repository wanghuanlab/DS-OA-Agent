FROM COMMIT TO LOG

工作已经做过一次，
不必再回忆一次。

从本地代码库读取真实工作轨迹，用 LLM 整理成中文日志，再通过禅道 HTTP 接口直接录入。准备、预览、提交，都在一个客户端内完成。

提交人筛选多任务拆分剩余工时续算

CORE CAPABILITIES

少一点重复填报，
多一点确定性。

01

自动检测提交，生成工作总结

选择代码库和日期后，自动识别 Git、Mercurial 或 SVN，按提交人筛选记录，再生成简洁的中文工作描述。普通目录会自动忽略。

GitHGSVN
02

无需打开禅道，VPN 连接即可填报

应用自动检测网络状态、登录并获取当前任务。确认预览后，通过 HTTP 接口直接录入日志，不再逐页寻找入口。

VPN登录任务
03

本地优先，配置透明可控

源代码公开可审查。禅道账号、LLM Key、代码库路径和任务关联均以 JSON 保存在当前用户目录，不会写入项目仓库。

在 GitHub 查看实现 ↗

REVIEW BEFORE SUBMIT

同一天，可以有多个任务。

日期、任务、工作描述和耗时都能直接编辑。应用同时展示任务剩余时间，提交前每一条都清清楚楚。

ONE LOCAL WORKFLOW

三步完成一周日志。

  1. 01准备

    连接禅道和 LLM，选择日期、代码库、关联任务与提交人。

  2. 02预览

    生成中文日志，按日期继续调整任务、描述和实际耗时。

  3. 03录入

    核对周期、任务数和总耗时后，由你确认并立即提交。

LOCAL BY DESIGN

代码公开。
配置留在本机。

所有个人配置都保存到操作系统的当前用户数据目录。项目仓库只提供不含凭据的示例配置，工作日志提交也只在你的电脑与已配置服务之间发生。
